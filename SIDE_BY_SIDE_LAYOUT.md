# 并排布局实现文档

## 功能更新
将"比特币地址查询"和"赞助支持"两个功能模块从上下堆叠改为并排显示，提升页面空间利用率和视觉效果。

## 布局设计

### 桌面端（> 968px）
- **左侧：比特币地址查询** - 占 60% 宽度 (1.2fr)
- **右侧：赞助支持** - 占 40% 宽度 (0.8fr)
- **间距：** 20px
- **对齐方式：** 两个模块高度自动对齐

### 平板端（< 968px）
- 自动切换为垂直堆叠布局
- 间距调整为 16px
- 每个模块占满全宽

### 移动端（< 768px）
- 保持垂直堆叠
- 赞助支持的二维码显示在地址下方
- 字体和间距进一步优化

## 技术实现

### HTML结构
```html
<div class="tools-row">
    <section class="btc-search-section">
        <!-- 比特币地址查询内容 -->
    </section>
    
    <section class="sponsor-section">
        <!-- 赞助支持内容 -->
    </section>
</div>
```

### CSS Grid布局
```css
.tools-row {
    display: grid;
    grid-template-columns: 1.2fr 0.8fr;  /* 60% : 40% */
    gap: 20px;
    padding: 20px 0;
    animation: fadeInUp 0.9s ease-out;
}
```

### 响应式断点
```css
@media (max-width: 968px) {
    .tools-row {
        grid-template-columns: 1fr;  /* 单列布局 */
        gap: 16px;
    }
}
```

## 视觉优化

### 尺寸协调
为了让并排显示更加协调，对两个模块的元素尺寸进行了统一调整：

#### 标题区域
- 图标大小：32px → 28px
- 标题字号：24px → 20px
- 描述字号：14px → 13px

#### 内容区域
- 容器内边距：32px 28px → 28px 24px
- 描述文字行高：1.6 → 1.5
- 整体更加紧凑，但仍保持良好可读性

#### 赞助支持地址
- 标签字号：15px → 14px
- 地址字号：15px → 13px
- 地址内边距：6px 12px → 6px 10px

### 高度对齐
- 两个容器使用 `height: 100%` 和 `flex` 布局
- 自动计算并保持相同高度
- 赞助支持内容垂直居中对齐

## 布局比例说明

选择 1.2:0.8 (60%:40%) 的比例原因：

1. **功能权重**：地址查询功能包含输入框和按钮，需要更多空间
2. **视觉平衡**：避免等宽布局显得呆板
3. **内容适配**：赞助支持内容相对简单，40% 宽度足够
4. **响应式友好**：在 1200px 左右的屏幕上显示效果最佳

## 用户体验提升

1. **空间利用**：一屏内展示更多内容，减少滚动
2. **视觉层次**：功能区域更加清晰，主次分明
3. **信息密度**：保持可读性的同时提升信息密度
4. **响应式适配**：自动适应不同屏幕尺寸

## 浏览器兼容性

- ✅ 现代浏览器（Chrome, Firefox, Safari, Edge）: 完美支持 Grid 布局
- ✅ IE11+: 支持 Grid 但可能需要前缀
- ✅ 移动浏览器: 完全支持响应式断点
